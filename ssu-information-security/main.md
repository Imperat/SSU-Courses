1. Основные концептуальные положения системы защиты информации.  Требования к защите информации и к системе защиты информации.
==============================================================================================================================
Основные концептуальные положения системы защиты информации.
------------------------------------------------------------
1. Обеспечение безопасности не может быть одноразвым актом.
2. Безопасность инормации может быть обеспечены ишь при комлексном использовании всего арсенала имеющихся средств.
3. Система защиты информации не может обеспечить требуемый уровень защиты информации без надлежащей подготовки пользователей.

Требования к защите информации
-------------------------------
1. Непрерывность
2. Плановость
3. Целенаправленность (от конкретных угроз)
4. Конкретной
5. Активной
6. Надёжной
7. Универсальной
8. Комплексной (на всех уровнях)

Требования к системе защиты информации
--------------------------------------
1. Чёткость определения полномочий и прав пользователей на доступ к определённым видам информации.
2. Предоставление пользователям минимально необходимых полномочий для выполнения работ.
3. Сведение к минимуму числа общих средств защиты информации.
4. Учёт попыток несанкционированного доступа.
5. Оценка степени конфиденциальности информации.
6. Контроль целостности средств защиты, немедленное реагирование на их выход из строя.


2. Концептуальная модель информационной безопасности.
=====================================================
Информационная безопасность - состощие защищённости информационной среды общества, обеспечение её формирование и развития в интересах государства, общества.

1. Объекты угроз - сведения о состоянии, составе, деятельности объекта защиты.
2. Угрозы - потенциально возможное событие, действие, процесс, явление, наносящее ущерб чьим-либо интересам.
3. Угрозы информационной безопасности - потенциальные действия, приводящие к искажению, уничтожению, копированию, блокировке доступа к информации, воздействие на компьютерные информационные системы.
4. Источники угроз - исполнители угрозы: люди, технические средства, внешняя среда.
5. Источники информации - люди, документы, публикации, технические носители, продукции отходных производств.
6. Несанкционированный доступ возникает за счёт утечки информации по техническим каналам, несанкционированному доступу к охраняемым сведениям.
7. Направления защиты: правовая, организационная, инженерно-техническая.
8. Способы защиты: всевозможные меры, пути, способы и действия, обеспечивающие предотвращение противоправных действий и преовтращающие несанкционированный доступ.
9. Средства защиты: физические, аппаратные, программные, криптографические методы.


3. Угрозы конфиденциальной информации.
======================================
Потенциальные или реальные действия по отношению к информационным ресурсам, приводящие к неправомерному овладению охраняемыми сведениями => моральный, материальный ущерб. (Ознакомление, Модификация, Разрушение).

Классы угроз
-------------
*Величина ущерба*
1. Предельный
2. Значительный
3. Незначительный

*Природа воздействие*
1. Естественное
2. Преднамеренное
3. Непреднамеренное

*Источник угроз*
1. Природа
2. Человек
3. Санкционированные программно-аппаратные воздействия.
4. Несанкционированные программно-аппаратные воздействия.

*Положение источника*
1. Вне контролируемой зоны
2. В пределах
3. В местах доступа к терминалам системы
4. Расположен в системе

*Характер нанесённого ущерба*
1. Материальный
2. Моральный

*По степени зависимости от активации системы*
1. Только в процессе работы
2. Бездействия
3. Вне зависимости от активации системы.

*Характер воздействия*
1. Активные (модификация)
2. Пассивные (без модификации)

*Способ доступа к ресурсам*
1. Прямой путь (хищение паролей)
2. Скрытный/нестандартный способ

*Место расположения информации*
1. Внешний накопитель
2. RAM
3. Канал связи
4. Диск

* По положению к объекту*
1. Внутренние
2. Внешние

Основные направления реализации угроз
-------------------------------------
1. Непосредственное обращение к объекту доступа.
2. Создание програ => обращение к объекту в обход метода защиты.
3. Модификация системы защиты
4. Внедрение закладок

Основные методы реализации угроз
---------------------------------
1. Определение типа и параметров носителей информации.
2. Получение информации о программно-аппаратной среде, типе, параметрах средств вычислительной техники, типе и версии ОС, составе ПО.
3. Получение информации о функциях, выполняемых системой.
4. Получение информации о применяемых системах защиты.
5. Определение способа представления информации.
6. Определение содержания данных на качественном уровне.
7. Определение содержания данных на семантическом уровне.
8. Использование специальных технических средств для перехвата наводок.
9. Уничтожение средств вычислительной техники и носителей информации.
10. Копирование носителей информации.
11. Хищение носителей информации.
12. НСД в обход или путём преодоления СЗИ с использованием спец. средств
13. НС превышение пользователем своих полномочий.
14. НС копирование ПО.
15. Перехват данных, передаваемых по линиям связи.
16. Визуальное наблюдение.
17. Раскрытие представления информации.
18. Внесение НС измененеий в программно-аппаратные компоненты системы и обрабатываемые данные.
19. Установка и использование нештатного аппаратного и ПО.
20. Заражение вредоносными программами.
21. Искажение информации.
22. Внедрение дезинформации
23. Проявление ошибок проектирования.

6. Парольные системы, угрозы безопасности ПС.
==========================================
Идентификатор (логин) - некоторая уникальная информация, позволяющая различать пользователей ПС.
Пароль - некоторая секретная информация, известная только пользователю и ПС, которая может быть запомнена пользователем и предъявлена ПС.
Ключ - аналог пароля, однако отличается тем, что не может быть легко запомнена пользователем (из-за большого объёма или отсутствия смысловой нагрузки) или не может быть введена с клавиатуры (из-за содержащихся в ней служебных символов, которые отсутствуют на клавиатуре)
Учётная запись пользователя - совокупность идентификатор + пароль
Идентификация - присвоение пользователям идентификаторов и проверка предъявляемых идентификаторов по списку присвоенных.
Аутентификация - проверка принадлежности пользователю предъявленного им идентификатора.

Способы аутентификации
-----------------------
1. По хранимой копии пароля
2. По хранимому хэшу пароля
3. По проверочному значению
4. Без передачи информации о пароле проверяющей стороне (доказательство с нулевым разглашением)
5. Использование пароля для получение криптоключа.

Угрозы безопасности ПС
----------------------
1. Подбор в интерактивном режиме.
2. Подсматривание.
3. Преднамеренная передача другому лицу.
4. Захват БД ПС
5. Перехват переданной по сети информации о пароле (повторное использование информации, восстановление доступа, модификация передаваемой информации с целью ввести в заблужение ПС, имитация действий ПС для введения в заблуждение пользователя)
6. Хранение информации о пароле в доступном месте
7. Внедрение закладок
8. Использование ошибок в ПО
9. Выведение из строя ПС
10. Обход ПС
11. Социальная инженерия
